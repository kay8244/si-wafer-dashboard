**Create `src/hooks/useAiInsight.ts`** (per D-01, D-02, D-03):

```typescript
'use client';

import { useState, useRef, useEffect, useCallback } from 'react';

interface UseAiInsightOptions {
  tab: string;
  buildContext: () => string | null;
  deps: unknown[];
}

interface UseAiInsightReturn {
  aiInsight: string | null;
  aiLoading: boolean;
  aiOpen: boolean;
  fetchAiInsight: (force?: boolean) => void;
  aiProps: {
    insight: string | null;
    loading: boolean;
    open: boolean;
    onToggle: () => void;
    onClose: () => void;
  };
}
```

Hook implementation rules:
- `buildContext` callback stays in each consumer component (per D-01 -- "context 빌드 로직은 각 컴포넌트에 남긴다")
- `deps` array triggers auto-refetch when panel is open (per D-02 -- "deps 배열을 받아 변경 시 자동 refetch")
- Use `useRef(false)` for `aiLoadingRef` to prevent concurrent fetches
- Use `useRef('')` for `prevDepsRef` to compare serialized deps via `JSON.stringify`
- `fetchAiInsight(force?)` logic: if not forced, toggle aiOpen state; if forced or first fetch, call `/api/ai-insight` POST
- Include `aiInsight` and `aiOpen` in `useCallback` dependency array for `fetchAiInsight` to avoid stale closures (Pitfall 2 from RESEARCH.md)
- `useEffect` for auto-refetch: compare `JSON.stringify(deps)` with `prevDepsRef.current`, refetch if changed while `aiOpen` is true
- `fetchAiInsight` MUST be in useEffect dependency array -- this is what eliminates the need for eslint-disable
- Error handling: `catch (err) { console.error('[ai-insight] ${tab} fetch failed:', err); }` (per D-10 pattern)
- Return `aiProps` bundle: `{ insight: aiInsight, loading: aiLoading, open: aiOpen, onToggle: () => fetchAiInsight(), onClose: () => setAiOpen(false) }`
