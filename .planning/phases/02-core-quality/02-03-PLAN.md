**Step 2: Create `src/lib/api-schemas.ts`**

Per D-07 (3 endpoints only) and D-08 (타입 + 길이 제한):

```typescript
import * as z from 'zod';

/** POST /api/ai-insight body schema */
export const aiInsightSchema = z.object({
  tab: z.string().max(50),
  context: z.string().max(10000),
});

/** GET /api/transcript query params schema */
export const transcriptSchema = z.object({
  customer: z.string().max(50),
});

/** GET /api/news query params schema */
export const newsSchema = z.object({
  queryKo: z.string().max(200),
  queryEn: z.string().max(200),
  companyName: z.string().max(100).optional(),
  context: z.string().max(200).optional(),
});

export type AiInsightInput = z.infer<typeof aiInsightSchema>;
export type TranscriptInput = z.infer<typeof transcriptSchema>;
export type NewsInput = z.infer<typeof newsSchema>;
```

Use Zod v4 import syntax: `import * as z from 'zod'` (NOT `import { z } from 'zod'` which is v3 style).
Use `.max()` for string length limits (NOT `.message()` which is v3 -- use `.error()` in v4 if custom error messages needed, but default messages are fine for this use case).
Do NOT use `.strict()` on objects -- Zod v4 uses `z.strictObject()` instead, but we want to allow extra fields to pass through silently.
