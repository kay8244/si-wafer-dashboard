Fix the 4th and final `eslint-disable-line react-hooks/exhaustive-deps` in the codebase.

**Current code (lines 47-52):**
```typescript
useEffect(() => {
  if (data) {
    updateOverlay(['SEC'], 'capa');
  }
}, [data]); // eslint-disable-line react-hooks/exhaustive-deps
```

**The issue:** `updateOverlay` is called inside the effect but not in the dependency array. However, `updateOverlay` is already defined as a `useCallback` with `[data]` as its dependency (per RESEARCH.md analysis). This means:
- When `data` changes, `updateOverlay` gets a new reference
- The effect should depend on both `data` and `updateOverlay`
- Since both change together (both depend on `data`), adding `updateOverlay` to deps does NOT cause extra re-runs

**Fix:** Change the dependency array to include `updateOverlay` and remove the eslint-disable comment:
```typescript
useEffect(() => {
  if (data) {
    updateOverlay(['SEC'], 'capa');
  }
}, [data, updateOverlay]);
```

Remove the `// eslint-disable-line react-hooks/exhaustive-deps` comment entirely.

**Verify after fix:** Run `npm run build` to confirm no infinite loop or build error.
