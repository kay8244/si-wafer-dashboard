Refactor `getSqliteDb()` in `src/lib/db.ts` to use a `globalThis`-cached singleton pattern (survives Next.js HMR in dev). Concrete changes:

1. Add a globalThis type extension and singleton accessor at the top of the SQLite section (after the `_seedDb` variable, around line 40):

```typescript
const globalForDb = globalThis as typeof globalThis & {
  _sqliteDb?: import('better-sqlite3').Database;
};

function getSqliteDb(): import('better-sqlite3').Database {
  if (!globalForDb._sqliteDb) {
    // eslint-disable-next-line @typescript-eslint/no-require-imports
    const Database = require('better-sqlite3') as typeof import('better-sqlite3');
    const dbPath = require('path').join(process.cwd(), 'data', 'dashboard.db');
    globalForDb._sqliteDb = new Database(dbPath);
    globalForDb._sqliteDb.pragma('journal_mode = WAL');
  }
  return globalForDb._sqliteDb;
}
```

2. Remove the `db.close()` calls in `queryAll()` (lines ~130 and ~135). The try/catch structure stays but `db.close()` must be deleted from both the success and error paths. Singleton connections must never be closed by query functions.

3. Keep the existing `_seedDb` / `getDb()` pattern unchanged -- it is used by seed scripts and already works as a singleton.

4. Keep ALL existing exports (`MetricRow`, `parseMeta`, `getPostgresUrl`, `isPostgres`, `ensureSchema`, `getDb`, `queryMetrics`, `queryAll`, `queryMetricsLike`, `queryMetricsIn`, `queryByCustomer`) with identical signatures.

5. The `DB_PATH` constant at line 7 can remain as-is (it is not used by the new singleton which computes path at call time).

Do NOT add a process.exit handler to close the DB -- better-sqlite3 handles cleanup on process exit automatically.
